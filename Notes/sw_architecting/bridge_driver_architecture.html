Bridge Driver & SVM Orchestrator

SW Architecture detail · child document of Motor Driver Architecture · Revision 0.1

Component Breakdown

svm_orchestrator
Coordinator
Responsibility
Receives a voltage vector command (V_alpha, V_beta) normalized to [-1, +1] from foc_u_guy. Determines the SVM sector, computes dwell times for the two active vectors and the zero vector, and derives per-phase duty cycles. Writes all three bridge_cmd_t structs atomically before any BridgeDriver consumes them.
Inputs
svm_cmd_t — V_alpha, V_beta (float, normalized), V_dc (for scaling)
Outputs
bridge_cmd_t[3] — duty cycle + enable for phases A, B, C. Written atomically via shadow register pattern before commit trigger.
Notes
Only instantiated for 3-phase motor types (BLDC, PMSM). Bypassed entirely for brushed DC (foc_u_guy writes directly to a single BridgeDriver) and stepper (separate orchestrator per winding pair). This is the sole component that has visibility across all three phases simultaneously — no other component should compute cross-phase duty relationships.
bridge_driver
Realtime · ×N instances
Responsibility
Owns one half-bridge. Accepts a bridge_cmd_t (duty, enable, brake). Applies dead time insertion. Writes high-side and low-side duty to PWM peripheral. Responds to gate kill signal from Fault_Responder by immediately disabling both outputs regardless of pending commands.
Inputs
bridge_cmd_t — duty (float 0–1), enable (bool), brake (bool)
gate_kill — direct fault line from Fault_Responder, checked each cycle before any PWM write
Outputs
PWM signals to LTC7066 HIN and LIN pins. No software output — purely a hardware actuator.
What it does NOT know
It has no knowledge of which phase it is, what motor topology is in use, or what any other bridge_driver instance is doing. All cross-phase coordination is the orchestrator's responsibility.

Atomic commit requirement: svm_orchestrator must write all three bridge_cmd_t structs and then issue a single commit before any BridgeDriver reads. If BridgeDriver[A] sees a new duty while B and C still hold old values, a spurious voltage vector is applied for one PWM cycle. Implementation: shadow register pattern — orchestrator writes to pending structs, single commit flag triggers all three reads simultaneously. Whether LEDC/RMT supports hardware-level sync across channels needs to be verified (see TODO below).

Reuse Across Motor Topologies

Motor Type Orchestrator bridge_driver instances Notes
BLDC / PMSM svm_orchestrator 3 (phases A, B, C) Full SVM sector switching. All instances coordinated.
Brushed DC none — direct 1 (H-bridge = 2 half-bridges, or just 1) foc_u_guy writes duty directly to bridge_driver. Simple PI on current. svm_orchestrator not instantiated.
Stepper stepper_orchestrator (TBD) 2 (winding A, winding B) Separate orchestrator handles microstepping phase relationships. bridge_driver reused unchanged.
Switched reluctance sr_sequencer (TBD) up to 4 Unipolar switching per phase. Simpler than SVM but requires rotor position gating. bridge_driver reused.
